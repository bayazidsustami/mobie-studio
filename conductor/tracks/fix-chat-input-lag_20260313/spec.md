# Specification: Fix Chat Input Lag and Selection Issue

**Track ID:** fix-chat-input-lag_20260313
**Type:** Bug
**Created:** 2026-03-13
**Status:** Draft

## Summary

The main chat prompt input in the GPUI frontend becomes significantly laggy when a user types continuously and the text exceeds two lines. Additionally, selecting all text (Ctrl+A/Cmd+A) only visually highlights the first line of text.

## Context

Mobie Studio uses a GPUI-based frontend. The chat input area is a core component of the user experience, allowing users to send goals to the AI agent. Performance and accurate rendering of standard text editing interactions (like text selection) are critical for the application's usability, especially when users input long, detailed prompts.

## Problem Description

**Steps to Reproduce:**
1. Open the Mobie Studio application.
2. Focus the main chat prompt input field.
3. Continuously type text until it wraps or exceeds two lines.
4. Attempt to type further (observe lag).
5. Press Ctrl+A (Windows/Linux) or Cmd+A (Mac) to select all text.

**Actual Behavior:**
- The input field becomes noticeably laggy when typing text beyond two lines.
- When attempting to select all text, only the first line is visually highlighted.

**Expected Behavior:**
- The input field should remain responsive and performant regardless of the text length or number of lines.
- Pressing Ctrl+A/Cmd+A should visually highlight the entirety of the text within the input field.

## Affected Areas

- Main chat prompt input component within the GPUI frontend (`src/ui/mod.rs` or related UI components).

## Acceptance Criteria

- [ ] Typing continuously for 5+ lines in the chat input shows no noticeable lag or frame drops.
- [ ] Pressing Ctrl+A/Cmd+A visually highlights all text across all lines in the chat input.
- [ ] Existing input functionality (sending messages, copy/pasting) remains unaffected.

## Dependencies

- `gpui` crate (UI Framework)

## Out of Scope

- Refactoring the entire UI layout.

## Technical Notes

The lag might be related to how the GPUI `TextInput` or custom text rendering component is handling layout recalculations or repaints when wrapping text. The selection issue suggests the selection bounds calculation might be failing to account for line breaks or vertical scroll offsets within the text element.
